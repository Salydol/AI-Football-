# ⚽ Football Intelligence Engine (FIE) v1.0.0

AI-платформа для анализа футбольных матчей: трекинг игроков, тактический анализ, предсказания, LLM-ассистент тренера и многое другое.

## Возможности

- 🎯 **Трекинг** — YOLO v8 + ByteTrack, координаты всех игроков и мяча в реальном времени
- ⚽ **Тактика** — расстановка, прессинг, компактность, тактические ошибки
- 🔮 **Предсказания** — исход матча (CatBoost), вероятность гола (xG)
- 👤 **Аналитика игроков** — усталость, цифровой паспорт, скаутинг
- 🧬 **Командная аналитика** — Team DNA, сравнение команд, анализ соперника
- 🎬 **Видео** — автоматическая нарезка клипов, поиск эпизодов на естественном языке (RU/EN)
- 🤖 **LLM** — Coach Assistant (Anthropic / OpenAI / Ollama / rule-based), AI Match Story
- 💡 **Explainable AI** — объяснения для каждого предсказания
- 🎓 **Академия** — трекер развития молодых игроков
- 📄 **PDF отчёты** — автоматический отчёт по матчу

## API Reference

### 🎯 Tracking — `/tracking`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/tracking/analyze-match` | Запустить полный трекинг матча: детекция игроков, трекер, метрики. Принимает путь к видео, возвращает покадровые данные всех игроков и мяча |
| POST | `/tracking/analyze-match/stream` | То же, но с потоковой передачей результатов (SSE) — для real-time отображения |

**Как работает:** загружаешь видео → YOLO детектирует игроков и мяч на каждом кадре → ByteTrack присваивает устойчивые ID → вычисляются скорость, ускорение, дистанция, направление.

---

### 📊 Player Ratings — `/ratings`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/ratings/player-ratings` | Рассчитать рейтинги игроков по данным матча. Возвращает физический / тактический / общий рейтинг для каждого игрока |

**Как работает:** принимает агрегированную статистику матча (дистанция, спринты, участие в прессинге, ошибки) → взвешенная формула → рейтинг 0–10.

### 📄 Reports — `/reports`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/reports/match-report` | Сгенерировать PDF отчёт по матчу. Возвращает готовый файл для скачивания |

**Как работает:** принимает полные данные матча → ReportLab генерирует профессиональный PDF с таблицами, рейтингами, тактическим анализом.

### ⚽ Tactical Analysis — `/tactical`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/tactical/analyze` | Полный тактический анализ: расстановка, прессинг, компактность, зоны, опасные атаки |
| POST | `/tactical/formation-only` | Только определение расстановки (быстро, без полного анализа) |
| POST | `/tactical/mistakes` | Найти тактические ошибки: потеря позиции, разрыв в обороне, провал прессинга, открытый коридор |

**Как работает:** принимает трекинговые данные (координаты всех игроков) → кластеризация по линиям → определение схемы → анализ прессинга (PPDA) → поиск нарушений структуры.

---

### 🔮 Predictions — `/prediction`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/prediction/match-outcome` | Вероятность победы / ничьей / поражения. Модель CatBoost на исторических данных |
| POST | `/prediction/goal-probability` | Вероятность гола в текущий момент с учётом позиции, угла, давления, xG |

**Как работает:** `match-outcome` — фичи из истории матчей → CatBoost → вероятности трёх исходов. `goal-probability` — позиция мяча, расстояние до ворот, угол, число защитников → xG формула.

### 👤 Analytics — `/analytics`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/analytics/fatigue-risk` | Оценить усталость и риск травмы игрока. Уровни: LOW / MEDIUM / HIGH / CRITICAL |
| POST | `/analytics/passport/add-match` | Добавить матч в цифровой паспорт игрока |
| GET | `/analytics/passport/{player_id}` | Получить цифровой паспорт: история, прогресс за сезон, тренды |
| GET | `/analytics/passport/{player_id}/history` | История всех матчей игрока |

**Как работает:** `fatigue-risk` — анализирует число спринтов, ускорений, падение скорости за матч → многофакторная модель риска. `passport` — save/load JSON на диск, агрегация за сезон.

### 🎬 Clipping — `/clipping`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/clipping/extract-clips` | Нарезать клипы из видео по списку событий (тайм-коды). Возвращает пути к файлам |
| POST | `/clipping/highlights` | Автоматически найти и нарезать хайлайты (топ события по confidence) |
| GET | `/clipping/clips` | Список всех сохранённых клипов |
| GET | `/clipping/clips/{filename}` | Скачать конкретный клип |

**Как работает:** принимает список событий с тайм-кодами → OpenCV нарезает видео с заданным буфером до/после события (default: 4 сек до, 3 сек после) → сохраняет MP4 файлы.

### 🧬 Team Analytics — `/analytics`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/analytics/team-dna/add-match` | Добавить матч в профиль Team DNA команды |
| GET | `/analytics/team-dna/{team_id}` | Получить тактический профиль команды (DNA вектор) |
| GET | `/analytics/team-dna/{team_id}/compare/{team_id_2}` | Сравнить стиль двух команд, получить similarity score |
| GET | `/analytics/scouting/radar/{player_id}` | Скаутинговый радар игрока: talent_score, сравнение с эталоном позиции |
| POST | `/analytics/scouting/find-similar` | Найти игроков с похожим профилем |
| POST | `/analytics/scouting/search` | Поиск игроков по критериям (позиция, минимальный рейтинг) |
| GET | `/analytics/scouting/compare/{player_id_1}/{player_id_2}` | Сравнить двух игроков по всем метрикам |

**Как работает:** `team-dna` — агрегирует прессинг, темп, территориальность, ширину атаки по матчам → строит вектор стиля → косинусное сходство для сравнения. `scouting` — позиционный эталон (нормы для CM, ST, CB и т.д.) → deviation score → talent_score.

### 🤖 LLM — `/llm`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/llm/coach-ask` | Задать вопрос тренерскому ассистенту на RU/EN. Анализирует данные матча и отвечает |
| POST | `/llm/tactical-report` | Сгенерировать полный тактический разбор матча |
| POST | `/llm/substitution-advice` | Рекомендации по заменам на основе усталости и рейтингов |
| POST | `/llm/match-story` | Сгенерировать AI нарратив матча в стиле спортивного журналиста |
| GET | `/llm/status` | Проверить какой LLM backend активен |

**Как работает:** данные матча → форматируются в текстовый контекст → передаются в LLM (Anthropic / OpenAI / Ollama) или rule-based движок → ответ на языке вопроса. Без API ключа — rule-based режим (бесплатно, offline).

**Бэкенды (приоритет):** Anthropic Claude → OpenAI GPT → Ollama (локально) → Rule-based.

### 🔍 Video Search — `/search`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/search/index-video` | Проиндексировать видео: запустить детектор событий и сохранить индекс |
| POST | `/search/index-from-events` | Построить индекс из готового списка событий (без детектора) |
| POST | `/search/query` | Поиск по видео на естественном языке: «все удары во втором тайме» |
| POST | `/search/query-and-clip` | Найти эпизоды + автоматически нарезать клипы за один запрос |
| GET | `/search/index/{video_id}` | Статистика существующего индекса |
| GET | `/search/list` | Список всех проиндексированных видео |

**Как работает:** `index-video` → EventDetector обрабатывает каждый кадр → сохраняет события с тайм-кодами в JSON. `query` → NL запрос парсится (rule-based или LLM) → фильтрация по типу события / тайму / минуте / игроку.

**Примеры запросов:**
- `"все удары в первом тайме"`
- `"опасные моменты после 75 минуты"`
- `"передачи игрока #7"`
- `"last 10 minutes tackles"`
- `"показать ошибки правого защитника"`

### 💡 Explainable AI — `/explain`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/explain/goal-probability` | Объяснить вероятность гола: почему именно такой % и какие факторы влияют |
| POST | `/explain/fatigue` | Объяснить уровень усталости игрока: спринты, падение скорости, нагрузка |
| POST | `/explain/tactical-mistake` | Объяснить тактическую ошибку: что произошло, почему проблема, как исправить |
| POST | `/explain/match-prediction` | Объяснить предсказание исхода матча с ключевыми факторами |
| POST | `/explain/team-dna` | Перевести DNA вектор в понятный тактический профиль |
| POST | `/explain/scouting` | Объяснить скаутинговую оценку игрока |

**Как работает:** принимает числовые данные → rule-based движок формирует структурированное объяснение на RU/EN → возвращает `verdict` (итоговый вывод) + `factors` (список причин) + `severity` (info / warning / critical).

**Пример:** вероятность гола 72% → `"ВЫСОКАЯ вероятность гола: 72.0%"` + факторы: `["Нет защитников между мячом и воротами", "Хороший угол удара (50°)", "Удар с близкой дистанции (9.0м)"]`
